# ЧАСТЬ I. ПРОДУКТ И КОНЦЕПЦИЯ

## 1.1 Что это

**«Новый уровень»** — веб-игра (PWA) для мобильного оператора **T2** (бывший Tele2). Казуальный таймкиллер в жанре бесконечного вертикального климбера (аналог Doodle Jump). Открывается по ссылке в браузере телефона, после первого захода работает офлайн.

**Слоган-якорь:** «Другие правила. Новый уровень» — игра буквально визуализирует слоган: игрок поднимается на новые уровни сети.

## 1.2 Зачем это T2 (бизнес-цели)

1. **Engagement:** вернуть абонента в приложение T2 каждый день (цель +15-20% DAU на сегменте игроков)
2. **Обучение продукту:** бустеры названы по реальным продуктам — игрок узнаёт линейку без рекламы
3. **Апсейл:** награды (ГБ, дни MiXX) стимулируют пользование услугами (+5-8% конверсия в MiXX)
4. **PR:** «оператор связи сделал игру, работающую без связи» — ироничный инфоповод под слоган
5. **Виральность:** лидерборд + шеринг рекордов + прямая ссылка без установки

## 1.3 Целевая аудитория

- Основная: абоненты T2, 18-45 лет, mass market, играют в транспорте/очередях
- Вторичная: друзья абонентов (приходят по ссылке), потенциальные MNP-переходы

## 1.4 Elevator pitch (для презентации)

> Это Doodle Jump во вселенной T2: прыгаешь по вышкам связи из 90-х в будущее, собираешь гигабайты, получаешь реальные награды на тариф. Работает в браузере без установки и — внимание — без интернета. Управление на выбор: наклон или свайпы, чтобы играть даже в трясущейся маршрутке. Другие правила. Новый уровень.

## 1.5 Столпы качества (по ним меряем каждый этап)

1. **Game feel** — прыжок должен приносить удовольствие сам по себе. Это фундамент; если прыгать не в кайф, остальное не имеет значения
2. **Честность** — игрок умирает только по своей вине, никогда из-за генератора
3. **Бренд** — любой кадр игры узнаваем как T2 без логотипа
4. **Надёжность** — офлайн работает всегда, прогресс не теряется никогда
5. **Лёгкость** — быстро грузится, мгновенно запускается, не тормозит на бюджетных телефонах

# ЧАСТЬ II. ГЕЙМДИЗАЙН (ПОДРОБНО)

## 2.1 Core loop

```
Старт → Прыжки вверх по платформам → Сбор кристаллов/бустеров
  → Рост высоты → Смена эпох (визуальная награда)
  → Падение → Game Over → Показ награды → Рестарт одним тапом
```

Сессия: 3-10 минут. Смерть не наказывает — мгновенный рестарт, кристаллы сохраняются.

## 2.2 Персонаж — «Сигнал»

- Абстрактный светящийся сгусток: ядро маджента `#FF3495`, белый шлейф-хвост
- Не мультяшный, не анимешный. Геометрия + свечение
- Прыгает автоматически при касании платформы (управляем только горизонталью — стандарт жанра)
- Анимации: idle (парение с покачиванием), прыжок (squash & stretch), полёт (вытягивание), падение (рагдолл + вращение)
- След: тонкий белый trail, исчезает за 0.5 сек

## 2.3 Физика (стартовые параметры для тюнинга)

```
Гравитация:            0.6 px/frame² (Matter.js: gravity.y = 1, scale подобрать)
Импульс прыжка:        -16 px/frame (вверх)
Импульс супер-прыжка:  -24 px/frame (трамплин-платформы, опция)
Горизонт. скорость:    макс 8 px/frame
Затухание горизонт.:   0.92 (инерция при отпускании)
Wrap по горизонтали:   ДА — вышел за левый край, появился справа (классика жанра)
Хитбокс персонажа:     круг r=20 px (при базовом размере спрайта 48x48)
Коллизия с платформой: только сверху и только при падении (velocity.y > 0)
                       — пролетаем платформу снизу насквозь
```

**Критично для game feel:** коллизия проверяется только когда персонаж летит ВНИЗ. Иначе игрок «бьётся головой» — это стандарт жанра, нарушать нельзя.

**Тюнинг физики — отдельная задача, не побочная.** Настройка «весёлого прыжка» делается итерациями на реальном устройстве: высота прыжка vs зазоры, инерция vs отзывчивость. Закладываем полноценный цикл: играем → крутим balance.json → играем. Критерий готовности: хочется сыграть «ещё разок» без всякого контента.

## 2.4 Управление (ДИФФЕРЕНЦИАТОР — на выбор игрока)

### Режим 1: Акселерометр
- API: `DeviceOrientationEvent` (свойство `gamma` — наклон влево/вправо)
- iOS 13+: требуется `DeviceOrientationEvent.requestPermission()` — вызывать по тапу (кнопка «Разрешить наклон»)
- Чувствительность: gamma ±30° = полная скорость; мёртвая зона ±3° (защита от дрожи рук)
- Настройка: Низкая (±45°) / Средняя (±30°) / Высокая (±20°)
- Калибровка «нуля» при старте партии (телефон уже наклонён — считаем это нулём)

### Режим 2: Тач/свайпы
- Вариант A: палец на экране → персонаж плавно движется к X пальца (следование, не телепорт)
- Вариант B: зоны — левая половина = влево, правая = вправо
- Делаем ОБА варианта, выбираем основной по плейтестам, второй оставляем в настройках
- Ключевая продажа: работает в транспорте, где акселерометр ловит тряску

### Переключение
- В настройках + быстрый тумблер на экране паузы
- Первый запуск: онбординг «Как удобнее играть?» — 2 карточки, выбор сохраняется
- Переключение доступно в любой момент, состояние в localStorage

## 2.5 Платформы (вышки связи) — 4 типа

| Тип | Метафора T2 | Поведение | Вид | Появление |
|---|---|---|---|---|
| **ВОЛС** (стабильная) | Оптика — надёжность | Обычная, вечная | Белый контур, чёткая | Вся игра, база |
| **РРЛ** (шаткая) | Радиорелейки — устаревают | Разрушается через 0.3с после касания; вариант — периодически мигает и «гаснет» | Маджента-контур, мерцает | Со 2-й эпохи, 10%→25% |
| **Движущаяся** | — | Ездит горизонтально, 1-3 px/frame | Белая со стрелками | С 3-й эпохи, 10%→20% |
| **Разовая** | — | Исчезает после одного прыжка | Полупрозрачная, пунктир | С 3-й эпохи, 5%→15% |

Ширина: 90 px (базовая) → 70 px (поздние эпохи). Высота 18 px.

## 2.6 Процедурная генерация

**Принцип:** не чистый рандом, а паттерны — pre-defined сегменты по 4-6 платформ, гарантированно проходимые.

```
Правила расстановки:
- Вертикальный зазор: 60-140 px (макс высота прыжка ≈ 180 px — запас на реакцию)
- Горизонтальный разброс: любой (есть wrap)
- Гарантированная ВОЛС каждые N платформ (страховка от unfair-смерти):
  эпохи 1-2 → каждая 3-я; эпохи 3-4 → каждая 4-я; эпоха 5 → каждая 5-я
- Паттерны: «лесенка», «зигзаг», «разрыв» (дальний прыжок), «лифт» (движущиеся),
  «минное поле» (РРЛ + разовые). Плюс запас на новые паттерны по итогам плейтестов
- Выбор паттерна взвешенно-случайный, веса зависят от эпохи
- Спавн: на 2 экрана выше камеры; чистка: на 1 экран ниже
- Валидатор проходимости: автотест прогоняет каждый паттерн на достижимость
```

**Сложность растёт с высотой:** доля шатких/разовых, ширина зазоров, скорость движущихся, ширина платформ. Всё в balance.json.

## 2.7 Эпохи (5 биомов) — детально

| # | Эпоха | Высота | Атмосфера | Палитра | Муз. стиль |
|---|---|---|---|---|---|
| 1 | **2G / 90-е** | 0–500 м | Панельки, старые антенны, тёплые окна, лёгкое VHS-зерно | Ч/Б + маджента-огоньки в окнах | Chiptune-ностальгия |
| 2 | **3G / нулевые** | 500–1500 м | Городские крыши, первые смарт-вышки, ранний цифровой шум | Ч/Б + электрик-блю мерцание | Ранний электро |
| 3 | **LTE / 2010-е** | 1500–3000 м | Облака, минимализм, ИНВЕРСИЯ: белый фон, чёрные силуэты | Белый + чёрный + маджента | Чистый электро |
| 4 | **4G+/5G** | 3000–5000 м | Стратосфера, футуристические башни | Чёрный + маджента + салатовый неон | Модерн-техно |
| 5 | **Будущее/6G** | 5000+ м | Космос, частицы кода, дата-абстракция | Чёрный + маджента + электрик-блю + белые частицы | Амбиент-электроника |

**Переход эпохи:**
- Кроссфейд фоновых слоёв 3-5 сек + вспышка
- Баннер «LTE достигнут. Можно ускоряться» (2 сек)
- Кроссфейд музыки
- Конвертация: 1 платформа ≈ 10 м высоты (читаемые цифры)

**Параллакс:** минимум 3 слоя (дальний ×0.2, средний ×0.5, ближний ×0.8 скорости камеры).

## 2.9 Препятствия

| Препятствие | Метафора | Поведение | Появление |
|---|---|---|---|
| **Помеха** (глитч-облако) | Сетевые помехи | Висит между платформами; касание = отброс вниз + потеря контроля 0.5с | С эпохи 2 |
| **Фейк-платформа** | Мошенничество | Выглядит почти как ВОЛС, растворяется без отскока; с SafeWall — подсвечена | С эпохи 3 |

Смерть ТОЛЬКО от падения за нижний край. Препятствия усложняют, но не убивают напрямую.

## 2.10 Кристаллы и экономика

```
Обычный кристалл: +1, спавн ~1 на 5 платформ, часто цепочками 3-5
Крупный кристалл: +5, редкий (~1 на 50 платформ), в труднодоступных местах

Очки (score) = высота в метрах. Кристаллы — отдельная валюта.

Экономика магазина (стартовые значения, тюнить по данным):
  500 кристаллов  → 1 ГБ на тариф
  1500 кристаллов → 3 дня MiXX
  3000 кристаллов → скидка 20% на тариф
  5000 кристаллов → 10 ГБ + месяц MiXX
Средний фарм: ~30-60 за сессию → награда раз в 10-20 сессий (retention-механика)
```

## 2.11 Смерть и Game Over

- Условие: ушёл за нижний край экрана (камера вниз не едет никогда)
- Щит/жизнь → батут-отскок от нижней границы + эффект разрушения щита
- Экран Game Over: высота (Halvar Stencil 96pt) → нарратив эпохи («Застрял на 3G. Даже обидно») → +кристаллы → рекорд (если побит — конфетти маджента) → кнопки: ЕЩЁ РАЗ (маджента, большая) / Меню / Поделиться

## 2.12 Мета-прогрессия и retention

- Личный рекорд (IndexedDB + синк)
- Лидерборд: глобальный топ-100, друзья, город
- Еженедельный сброс + призы топ-10
- Магазин наград (кристаллы → реальные продукты)
- Достижения: «Первые 1000 м», «Пережил 10 РРЛ», «Все бустеры за сессию» и т.д.
- Сезонные события (Новый год, день связиста) — после запуска

## 2.13 Tone of voice (тексты в игре)

Дерзко, коротко, без канцелярита:

| Контекст | Текст |
|---|---|
| Переход на 3G | «3G. Уже что-то» |
| Переход на LTE | «LTE достигнут. Можно ускоряться» |
| Переход на 5G | «5G. Почти космос» |
| Переход на 6G | «Будущее. Ты в нём первый» |
| Бустер Гигабэк | «Гигабэк качает» |
| Разрушение РРЛ | «Вышка погасла. Бывает» |
| Game Over (низко) | «Даже 2G так низко не падал» |
| Game Over (рекорд) | «Новый уровень. Буквально» |
| Офлайн-режим | «Сети нет. Игра есть» |

❌ Запрещено: «Вы успешно...», «К сожалению...», «Поздравляем! Вы...»

## 3.6 Анимации и микро-интеракции

- Прыжок: anticipation → stretch → squash
- Кристалл: вспышка частиц + «+1» флоат
- Бустер: радиальная волна + аура

## 4.3 Архитектура игрового цикла

```typescript
app.ticker.add((ticker) => {
  const dt = ticker.deltaMS;
  inputSystem.update();               // акселерометр/тач → velocity.x
  Matter.Engine.update(engine, dt);   // физика
  syncSpritesToBodies();              // Matter → Pixi позиции
  spawnerSystem.update(cameraY);      // генерация выше, чистка ниже
  collisionSystem.update();           // кристаллы, бустеры, препятствия
  cameraSystem.update();              // только вверх
  scoreSystem.update();               // высота, эпохи, переходы
  hudStore.setState({...});           // Zustand → React HUD (throttle 100мс)
});
```

- Камера: игрок не ниже 60% высоты экрана; вниз не едет
- React рендерит только UI; игровой мир — целиком PixiJS, ноль re-render на кадр
- One-way платформы: проверка `velocity.y > 0` + AABB (можно без Matter-коллизий — чистая математика, проще контролировать)

## 4.5 PWA / офлайн (ключевая фича)

**Концепция:** первый заход онлайн → всё кэшируется → работает без интернета, как динозаврик Chrome, только полноценная игра. Прогресс локально, при связи — фоновый синк.

```
Service Worker (Workbox через vite-plugin-pwa):
- precache: HTML, JS, CSS, шрифты, спрайты, UI-звуки
- runtime: CacheFirst для музыки эпох (lazy)
- navigateFallback: index.html

manifest.webmanifest:
{ name/short_name: "Новый уровень", display: "fullscreen",
  orientation: "portrait", background_color: "#000000",
  theme_color: "#FF3495", icons: [192, 512 + maskable] }

Обязательно:
- navigator.storage.persist() при старте
- beforeinstallprompt → своя кнопка «Установить»
- Update-toast при новом SW
- Синк: Background Sync API (Android) + синк-при-открытии (iOS)
- Дедупликация сессий по UUID; конфликты: рекорд=max, кристаллы=сумма
```

**Известные ограничения:**
- iOS: лимит ~50МБ, кэш чистится после 7 дней неактивности, нет Background Sync
- Первый заход требует интернета
- Очистка данных браузера убивает локальный прогресс → частый синк
- Вес: цель 5-8 МБ первичной загрузки (WebP, сжатый звук, lazy-музыка)

## 4.6 Бэкенд (полноценный, без моков)

```
API (REST):
POST /api/sessions/sync    — батч сессий с дедупликацией по UUID
GET  /api/leaderboard      — топ-100 (+фильтры friends/city)
GET  /api/profile          — профиль, баланс кристаллов
POST /api/rewards/redeem   — обмен кристаллов на награду
GET  /api/config           — balance.json с сервера (OTA-тюнинг без релиза)

Anti-cheat:

- Серверная валидация физической возможности цепочки
- Аномалии не попадают в лидерборд и не конвертируются в награды
- Rate limiting на начисления
```
