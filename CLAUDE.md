> Контекст проекта для Claude Code. Полная спецификация — в `docs/master_konspekt.md`. Читай её перед крупными задачами (геймплей, генерация, офлайн, дизайн-система).

## Что за проект

**«Новый уровень»** — веб-игра (PWA) для мобильного оператора **T2** (бывший Tele2). Казуальный таймкиллер, бесконечный вертикальный climber в стиле Doodle Jump. Персонаж-«сигнал» прыгает по вышкам связи вверх, проходя эпохи сетей 2G → 3G → LTE → 5G → Будущее. Открывается по ссылке в браузере, работает офлайн.

**Слоган:** «Другие правила. Новый уровень».

## Принцип работы

- **Без дедлайнов. Приоритет — качество.** Каждый этап закрывается по критериям готовности (DoD из мастер-конспекта, Часть V), а не по времени. Ничего не режем ради скорости.
- **Сначала game feel.** Прыжок должен приносить удовольствие сам по себе, до всякого контента. Физику тюним итерациями на реальном устройстве.
- **Честность к игроку.** Смерть только по вине игрока, никогда из-за генератора. Генерация — паттернами с валидатором проходимости, не чистый рандом.

## Стек

```
UI:          React 18 + TypeScript 5 (strict) + Vite 5
Рендер игры: PixiJS 8 (WebGL)
Физика:      Matter.js 0.19
Состояние:   Zustand
PWA:         vite-plugin-pwa + Workbox
Хранение:    IndexedDB (idb) + Cache Storage + localStorage
Тесты:       Vitest + Playwright
```

Игровой мир рендерит целиком PixiJS через ticker. React — только UI (меню, HUD, экраны). Никаких re-render React на игровой кадр. HUD обновляется через Zustand с троттлингом (≤10/сек).

## Структура

```

├── game/
│   ├── engine/   # PixiJS Application, ticker, resize
│   ├── entities/ # player, platform, crystal, booster, obstacle

│   ├── scenes/   # эпохи epoch1..epoch5, transitions

│   └── controls/ # accelerometer, touch, controls-manager
├── features/     # menu, shop, leaderboard, settings, onboarding
├── shared/       # ui (фирстиль T2), storage, audio, assets
└── pwa/          # manifest, SW, install-prompt, update-toast
```

**Правило:** все числа баланса (физика, спавн, экономика) — только в `src/game/config/balance.json`. Не хардкодить в логике.

**Правило (стили DOM/UI — БЭМ):** весь DOM/UI пишем по методологии **БЭМ** — классы `block__element--modifier`, стили в отдельном `*.css` рядом с модулем (пример: `src/game/controls/pauseMenu.css` → блок `pause`). Никаких инлайн-`style`/`cssText` в компонентах; состояния переключаем классами-модификаторами. Цвета — из токенов `:root` (`--t2-*`, `--surface-*`), не хардкодить hex. Глобалка (ресеты тегов, палитра-токены в `index.css`) по канону остаётся вне блоков. **БЭМ применим только к DOM** — игровой мир на PixiJS-канвасе (сущности = `Graphics`, не HTML) под БЭМ не подпадает.

## Ключевые правила геймплея (не нарушать)

- **One-way платформы:** коллизия только сверху и только при падении (`velocity.y > 0`). Снизу пролетаем насквозь.
- **Камера только вверх.** Вниз не едет никогда. Игрок держится не ниже 60% высоты экрана.
- **Горизонтальный wrap:** вышел за левый край — появился справа.
- **Автопрыжок** при касании платформы. Управляем только горизонталью.
- **Гарантированная ВОЛС-платформа** каждые N платформ (страховка от unfair-смерти).

## Управление (ключевая фича — на выбор игрока)

- **Акселерометр:** DeviceOrientation `gamma`, мёртвая зона ±3°, на iOS 13+ запрос permission по тапу, калибровка нуля при старте партии.
- **Тач:** следование за пальцем ИЛИ зоны лево/право. Работает в транспорте, где акселерометр ловит тряску.
- Переключение в настройках и на паузе. Выбор в localStorage. Онбординг при первом запуске.

## Фирстиль T2 (СТРОГО)

```css
--t2-black:   #000000;   /* фон, текст */
--t2-white:   #FFFFFF;   /* фон, текст */
--t2-magenta: #FF3495;   /* акцент, макс 20%, НИКОГДА не для текста */
/* доп (макс 20%, не миксовать): lime #C4F500, electric #1F3FFF, blue #0050FF */
```

- **НИКАКОГО жёлтого** — это старый Tele2 и Билайн. У T2 его нет.
- Текст и иконки — только чёрный или белый.
- Шрифты: Halvar Breitschrift (заголовки), Rooftop (текст), Halvar Stencil (счётчик высоты, score — фишка бренда). Fallback: Manrope ExtraBold / Inter / Big Shoulders Stencil.
- UI — бенто-сетка. Минимализм, без glassmorphism/градиентов/теней-глубин.
- Тон текстов: дерзко, коротко, без канцелярита. «LTE достигнут. Можно ускоряться», не «Вы успешно перешли...».

## Производительность

- Цель: 60 fps на iPhone SE 2020 / Redmi Note 9.
- ≤50 активных объектов, object pooling для платформ/кристаллов/частиц.
- Атлас спрайтов WebP, ParticleContainer для частиц, resolution = min(devicePixelRatio, 2).

## PWA / офлайн (ключевая фича)

Первый заход онлайн → всё кэшируется → работает без интернета. Прогресс в IndexedDB, при связи — фоновый синк. `navigator.storage.persist()` обязательно. manifest: fullscreen, portrait, theme #FF3495, bg #000000. Установка через beforeinstallprompt (своя кнопка).

**Этап 2 (Геймплейная глубина) — в работе.** Инкремент 1: **3 типа платформ** (`entities/platform.ts`: vols/rrl/moving; разовая убрана — дублировала РРЛ по механике) — РРЛ (`drawRrlShatter`): idle — маджента-контур + faint fill + **тонкая дрожь** (сдвиг view ~0.8px, косметика, коллизия по `p.x`) + лёгкое мерцание alpha; касание → распадается на **3 падающих осколка** за `collapseMs` (0.3с), затем деактивация. Коллизия пропускает уже разрушающуюся РРЛ. `trembleAmp` в конфиге. Движущаяся (`drawMovingChevrons`): белая база + **марширующие чёрные шевроны** (бесшовный конвейер: паттерн от −step до width+step, клип **маской** `p.maskG` по силуэту; сдвиг = дистанция платформы `animT += vx·dt` по модулю `chevronStepPx`; направление через **плавный `dirVisual`** (−1..1, едет к знаку `vx` за `chevronFlipSec`): при развороте стрелки складываются через вертикаль, марш замедляется до нуля и реверсится синхронно — без скачка. Марш замедлен `chevronSpeedFactor`. Перерисовка per-frame. Маска — Graphics-ребёнок view, включается только для moving. Спавнер (`systems/spawner.ts`): выбор типа по весам (`platforms.typeWeights`) + **гарантированная ВОЛС каждые N** (`spawn.guaranteedVolsEveryN`, страховка от нечестной смерти) + **ramp сложности по высоте** (`spawn.hazardStartMeters/hazardFullMeters`: <80м чистая ВОЛС, опасные набирают вес к 500м). Проверено: достижимость честная (perfect-x подъём 1558м без тупиков), ramp 0%→~28%.

Инкремент 2: **кристаллы** (`entities/crystal.ts`, `systems/crystals.ts`) — обычный (+1, цепочками 3–5 ~каждые 5 платформ по высоте), крупный (+5, ~каждые 50, у края). Сбор пролётом (окружность-окружность). Маджента-ромб. Счётчик справа-сверху (иконка+число), кнопка паузы сдвинута вниз (top 58px). **Landing-bounce (game feel):** при касании ВОЛС/движущейся платформа проседает вниз и пружинит (`bounceOffset` — затухающая синусоида, вниз ~7px → перелёт вверх → затухание за `bounce.durationSec` 0.6с). Только косметика `view.y` (коллизия/физика по `p.y` не трогаются); `p.bounceT` ставится в 0 при касании в createGame, крутится в spawner.update. РРЛ/фейк не пружинят.

Инкремент 4: **препятствия** (2.9). **Помеха** (`entities/obstacle.ts`, `systems/obstacles.ts`) — глитч-облако в воздухе выше `obstacles.interference.startMeters` (500м/эпоха 2); касание = отброс вниз (`knockbackVy`) + потеря управления (`controlLockSec`, обрабатывается в блоке ввода createGame через `controlLockSec`); попадание одноразовое. Визуал — **датамош-столбцы** (`drawObstacle`, концепт A2, per-frame по `animT`): вертикальные маджента-полосы ездят внапопад + микро-дрожь облака + «жёсткий сбой» раз в `glitchCycleSec` + мигающие белые sync-линии; фаза десинхронизирована при спавне. **Фейк-платформа** — тип `fake` в спавнере (гейт `obstacles.fake.startMeters` 1500м/эпоха 3, вес в `typeWeights`), визуал — **голограмма** (`drawFakeHologram`, per-frame по `p.animT`): бо́льшую часть цикла цельная белая (не отличить от ВОЛС), затем контур двоится (маджента-призрак ↖ + белый ↘, анаглиф) и схлопывается назад; `fake.shimmerSec` тюнит период, фаза десинхронизирована при спавне. При касании растворяется без отскока (проваливаешься). **Фейк — ДЕКОЙ**: спавнится парой с настоящей ВОЛС на том же уровне (в разных половинах экрана, `decoyPositions`) — иначе фейк создавал недостижимый двойной зазор (баг честности: real→fake→real ~260px > прыжка ~205px). Теперь опорный уровень всегда есть, макс зазор между опорами ≤ gapMax. Слой `obstacleLayer`. Смерть по-прежнему только от нижнего края. Проверено: фейк проваливает без отскока, помеха отбрасывает+блокирует, честный путь есть (perfect-x по не-фейкам 1573м, 0 смертей).

Инкремент 5: **бустеры** (2.8) — каркас (`entities/booster.ts`, `systems/boosters.ts`): сбор пролётом, per-type каданс (`spawnEvery` платформ), иконки-плейсхолдеры (двойной кристалл/гексагон-M/щит) парят+пульсируют, слой `boosterLayer`. Проверено: спавн/сбор, таймер тикает, ×2, аура. **×2 очки отложено** (нужен отдельный счётчик score для Game Over — сейчас score=высота, ретроактивно ×2 некорректно). **Инкремент B готов:** MiXX-щит = **заряды спасения** (`rescueCharges`, cap `rescueMax`=2); сбор → +1. Падение за нижний край: если заряд есть — тратится, **батут вверх** (`vy=-jumpVel*rescueBounceFactor` 1.5) вместо смерти; иначе reset. HUD: кольцевые таймеры активных timed-бустеров стопкой (Гигабэк маджента + SafeWall электрик-блю, мини-иконки), заряды-гексагоны над ними; аура приоритет Гигабэк→SafeWall(заливка)→щит(гекс-пузырь). **Вечные минуты убраны — дублировали MiXX-щит по механике** (оба давали +1 заряд); осталось **3 бустера с разными эффектами** (Гигабэк ×2-кристаллы / MiXX-щит спасение / SafeWall иммунитет+reveal). Проверено: сбор/таймер, иммунитет, reveal-рамка. **Все 3 бустера готовы.**

Инкремент 6: **валидатор проходимости** (Vitest, первые тесты в проекте). Чистая логика генерации вынесена из спавнера в `systems/generation.ts` (`PlatformPlanner` — без PixiJS, инъекция RNG `makeRng` mulberry32) — **единый источник правды**: спавнер только рисует placements, валидатор проверяет решения. `generation.test.ts` гоняет 50 сидов × ~3000м и проверяет: (1) на каждом уровне есть опора (фейк никогда не один — декой-инвариант), (2) макс зазор между опорами ≤ высоты прыжка (нет недостижимых разрывов), (3) гарантия ВОЛС (ограничен разрыв между ВОЛС-уровнями), (4) жадный климб достигает вершины без тупиков; + не-вакуумный тест (детектит фейк без декой). Это автомат-версия ручного perfect-x автопилота — поймал бы регресс честности. `npm test` (Vitest 2.x — на Node 20.10 rolldown-сборка Vitest 4 не идёт). Живой спавнер после рефактора идентичен (подъём 1121м, 0 смертей, пул ~22 платформы). **Опоры = vols/rrl/moving, фейк — нет.**

**Этап 4 (Звук) — в работе.** Инкремент 1: **процедурный аудио-движок** (`shared/audio/audioManager.ts`, синглтон `audio`) — весь SFX синтезируется на Web Audio (осцилляторы/шум), НОЛЬ файлов-сэмплов (в духе zero-asset проекта, офлайн-чистота). Контекст создаётся/резюмится по первому жесту (глобальный pointerdown/keydown — автоплей-политика). Шина: master(0.5)→sfxBus + musicBus (задел под музыку). SFX: `jumpVols` (мягкий блип, тихий+рандом высоты — звучит на КАЖДОМ касании, ~2-3/сек, не бесит), `jumpRrl`+`shatter` (отскок + крошево), `crystal(step)` (колокольчик, высота растёт по цепочке — арпеджио через `crystalCombo` в createGame), `booster(base)` (свелл, оттенок под тип), `rescue` (batут-boing), `glitch` (помеха), `fake` (провал), `death` (нисходящий power-down), `epochChord(id)` (мажор-аккорд со свеллом, корень растёт с эпохой — «вау-момент» DoD). Живой тумблер «Звуки» в настройках → `audio.setSound()`. Проверено: контекст running, все 10 SFX без ошибок, натуральная физика даёт jumpVols×22 + crystal-комбо + epochChord на переходе + death, мьют работает. **Звук проверен функционально (плумбинг), НЕ на слух** — тембры/громкость/«не бесит на 10-й минуте» тюнить пользователю на устройстве. `window.__audio` в dev. Инкремент 2 — **хаптика** (`shared/audio/haptics.ts`, синглтон `haptics`, `window.__haptics` в dev): Vibration API, три силы — tap 10мс (кристалл), hit 30мс (РРЛ-разрушение/помеха/бустер/спасение), heavy 80мс (смерть/переход эпохи). Проверено шпионом на navigator.vibrate: [10,30,80], мьют, кристаллы→10, смерть→80. Осталось по Этапу 4: музыка петель эпох + кроссфейд, микс-полировка, аудит звука на слух на устройстве.

**DEV-хук:** в dev-сборке доступен `window.__game` (`state()`, `step(n)`, `pause/resume`, `player`, `spawner`, `input`) — для отладки и тюнинга. В прод не попадает.

**Физика в реальном времени (frame-rate independent):** скорости в px/сек, ускорение px/сек²; интегрируем semi-implicit Euler фиксированным шагом `dtSec = 1/simHz`. Время кадра берём из `performance.now()` (НЕ `ticker.deltaMS`), аккумулятор + MAX_STEPS. Скорость игры не зависит НИ от FPS экрана, НИ от `simHz` (проверено wall-clock: 1.28М вызовов frame() за 300мс → 27 шагов = 90/сек). **Два независимых рычага тюнинга в balance.json: `jump.heightPx` (высота) и `jump.riseSec` (темп).** gravity/jumpVel выводятся из них. `physics.maxHorizontalSpeed` в px/сек, `input.followGainPerSec` — gain follow.

**Прим.:** rAF в фоновой вкладке браузер-автоматизации заморожен, поэтому логику проверяем через `__game.step(n)` (шаги simulate) или `__game.advance(ms)`, а визуал — на реальном устройстве по LAN-URL.
